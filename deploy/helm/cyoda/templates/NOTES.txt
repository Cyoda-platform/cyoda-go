/metrics auth:
  The chart forces CYODA_METRICS_REQUIRE_AUTH=true — /metrics on port
  9091 requires `Authorization: Bearer <token>`. Manual scrape test:
    kubectl -n {{ .Release.Namespace }} port-forward svc/{{ include "cyoda.fullname" . }} 9091:metrics &
    BEARER=$(kubectl -n {{ .Release.Namespace }} get secret {{ include "cyoda.metricsBearerSecretName" . }} \
      -o jsonpath='{.data.{{ .Values.monitoring.metricsBearer.existingSecretKey }}}' | base64 -d)
    curl -H "Authorization: Bearer $BEARER" http://localhost:9091/metrics | head

WARNING: do not set any chart-managed env via extraEnv — the chart sets
these and Kubernetes rejects duplicate env entries. This includes the
credentials and the cluster identity / tuning vars:
  CYODA_POSTGRES_URL / CYODA_POSTGRES_URL_FILE
  CYODA_JWT_SIGNING_KEY / CYODA_JWT_SIGNING_KEY_FILE
  CYODA_HMAC_SECRET / CYODA_HMAC_SECRET_FILE
  CYODA_BOOTSTRAP_CLIENT_SECRET / CYODA_BOOTSTRAP_CLIENT_SECRET_FILE
  CYODA_METRICS_BEARER / CYODA_METRICS_BEARER_FILE
  CYODA_METRICS_REQUIRE_AUTH
  CYODA_ADMIN_BIND_ADDRESS, CYODA_ADMIN_PORT
  CYODA_NODE_ID, CYODA_NODE_ADDR, CYODA_GOSSIP_ADDR, CYODA_SEED_NODES
  CYODA_STORAGE_BACKEND, CYODA_POSTGRES_AUTO_MIGRATE
  CYODA_IAM_MODE, CYODA_REQUIRE_JWT

To override credentials, change the referenced existingSecret — do not
add an extraEnv entry. extraEnv is for operator-owned additions like
OTel config, feature flags, and custom app env.
